# Future Work: FastEndpoints & Minimal API HTTP Endpoint Extraction

**Date:** 2026-04-01
**Status:** Proposed
**Scope:** C# HTTP endpoint detection

## Problem

The C# HTTP endpoint extractor (`csharp_http_extractor.py`) only detects the traditional ASP.NET Core controller pattern:

```csharp
[ApiController]
[Route("api/[controller]")]
public class TodosController : ControllerBase
{
    [HttpGet("{id}")]
    public async Task<Todo> Get(int id) { ... }
}
```

Two increasingly popular C# patterns produce zero Endpoint nodes and zero SERVES edges:

1. **FastEndpoints** — endpoint-per-class with `Configure()` method
2. **Minimal APIs / `IEndpointGroup`** — `Map()` with `MapGet`/`MapPost` calls

Both patterns were observed in the [CleanArchitecture](https://github.com/jasontaylordev/CleanArchitecture) reference project (0 Endpoint nodes after indexing).

## Pattern 1: FastEndpoints

```csharp
public class CreateTodo : Endpoint<CreateTodoRequest, CreateTodoResponse>
{
    public override void Configure()
    {
        Post("/api/todos");
        AllowAnonymous();
    }

    public override async Task HandleAsync(CreateTodoRequest req, CancellationToken ct)
    {
        // handler logic
    }
}
```

**Detection strategy:**
- Detect classes inheriting from `Endpoint<T>`, `Endpoint<TReq, TRes>`, `EndpointWithoutRequest`, or similar FastEndpoints base types via tree-sitter `base_list` node
- Find the `Configure()` method declaration in the class body
- Walk its body for `invocation_expression` nodes matching verb method names: `Get()`, `Post()`, `Put()`, `Delete()`, `Patch()`
- Extract the first string argument as the route
- Handler is the endpoint class itself (or `HandleAsync` method)

**Verb mapping:**
```python
_FASTENDPOINTS_VERB_MAP = {
    "Get": "GET", "Post": "POST", "Put": "PUT",
    "Delete": "DELETE", "Patch": "PATCH",
}
```

## Pattern 2: Minimal APIs with IEndpointGroup

This is what CleanArchitecture actually uses:

```csharp
public class TodoItems : IEndpointGroup
{
    public static void Map(RouteGroupBuilder groupBuilder)
    {
        groupBuilder.MapPost(CreateTodoItem);
        groupBuilder.MapPut(UpdateTodoItem, "{id}");
        groupBuilder.MapDelete(DeleteTodoItem, "{id}");
    }

    [EndpointSummary("Create a new Todo Item")]
    public static async Task<Created<int>> CreateTodoItem(ISender sender, CreateTodoItemCommand cmd)
    { ... }
}
```

**Detection strategy:**
- Detect classes implementing `IEndpointGroup` via tree-sitter `base_list`
- Find the `Map(RouteGroupBuilder)` method
- Walk its body for `invocation_expression` nodes on the group builder: `MapGet`, `MapPost`, `MapPut`, `MapDelete`, `MapPatch`
- First argument is a method reference (identifier) — resolve to the method in the same class
- Optional second argument is a route segment string
- Route prefix may come from `groupBuilder.MapGroup("/api/todos")` in the registration code (harder to trace — may require cross-file analysis or accept partial routes)

**Verb mapping:**
```python
_MINIMAL_API_VERB_MAP = {
    "MapGet": "GET", "MapPost": "POST", "MapPut": "PUT",
    "MapDelete": "DELETE", "MapPatch": "PATCH",
}
```

## Implementation Plan

All changes are localized to `src/synapps/indexer/csharp/csharp_http_extractor.py`. The existing `HttpExtractionResult`, `HttpEndpointDef`, `HttpPhase`, and Endpoint node schema are reused as-is.

| Step | What | Notes |
|------|------|-------|
| 1 | Add FastEndpoints base class detection | Walk `base_list` for known base types |
| 2 | Add `_parse_configure_method()` helper | Find `Configure()`, extract verb calls + routes |
| 3 | Add Minimal API `IEndpointGroup` detection | Walk `base_list` for `IEndpointGroup` |
| 4 | Add `_parse_map_method()` helper | Find `Map()`, extract `MapGet`/`MapPost` calls + method references |
| 5 | Route normalization | Existing `normalize_route()` handles most cases; ensure leading `/` |
| 6 | Test fixture | New fixture with both patterns, or extend SynappsTest |
| 7 | Integration tests | Verify Endpoint nodes + SERVES edges for both patterns |

## Complexity

**FastEndpoints:** Straightforward — similar to existing controller detection but checking inheritance instead of `[ApiController]` attribute, and reading `Configure()` body instead of method-level `[HttpGet]` attributes.

**Minimal APIs:** Slightly harder — method references in `MapPost(CreateTodoItem)` need resolution to the actual method's `full_name` in the same class. Route prefix from `MapGroup()` registration requires either cross-file tracing or accepting partial routes.
